How keywords work: These keywords are what the crawler searches for. PST Keywords trigger crawls across Reddit, Google News, Quora, Medium and 10+ more sources. Competitor names are tracked across the same sources and shown in the Competitors tab.
🔴 PST / Polaris Keywords — used in crawler.py BRAND_QUERIES
Tip: Use quotes around exact phrases. e.g. "Polaris School of Technology"
🏆 Competitor Keywords — tracked across all sources
After changing keywords: You need to update crawler.py with the new BRAND_QUERIES list and re-run the GitHub Action for changes to take effect. The dashboard shows a copy of your current keywords for reference.
To update crawler.py keywords:
1. Go to your GitHub repo → scripts/crawler.py
2. Find BRAND_QUERIES = [...] (line ~16)
3. Edit the list to match your PST keywords above
4. Commit → Actions will auto-run with new keywords
My Content tab shows YOUR manually logged content from Google Sheets. All fields are optional — only fill in tabs you actually maintain.
How to publish your Sheet as CSV:
Google Sheets → File → Share → Publish to web → select the tab → CSV → Publish

Sheet ID = the long code in your URL: /d/[THIS]/edit
GID = the number after #gid= in the tab URL (0 = first tab)

How to Know If It's Crawling
3 places to check
▼
⬤
1. This dashboard — crawl dot (top nav)
Green = crawled <3h ago • Amber = 3-8h • Red = stale >8h. Click the dot to open this debug panel.
📈
2. GitHub repo — Actions tab
Go to your repo → Actions → Polaris ORM Crawler. Green checkmark = success. Red X = failed. Click any run to see full logs.
📦
3. GitHub repo — data/ folder
Check data/summary.json — the last_crawled timestamp tells you exactly when it last ran. Commit history shows each crawl.
