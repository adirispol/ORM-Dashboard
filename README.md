## What This System Does

This is a full **Online Reputation Management (ORM) and Social Listening** platform for Polaris School of Technology, built entirely on free infrastructure. It runs automatically every 30 minutes and gives you:

| Feature | Details |
|---------|---------|
| **Brand Presence Score (BPS)** | 0–100 score (A+/A/B/C/D grade) showing your ORM health |
| **Multi-platform crawling** | Reddit · Quora · Medium · YouTube · News · Portals · Social · Web |
| **Real engagement metrics** | Reddit score/upvotes/comments, YouTube views/likes/comments, estimated Quora impressions, Medium reads |
| **Claude AI sentiment** | Every mention scored positive/negative/neutral/mixed using Claude Haiku |
| **YouTube comment crawl** | Deep crawl of all Polaris-related YouTube videos (checkpoint-based, ~5 days for full coverage) |
| **Competitor intelligence** | 8 Bangalore BTech colleges tracked: Scaler, Newton, PES, RVCE, BMS, Alliance, Jain, CMR |
| **Actionables engine** | Auto-generated prioritized next steps from crawl data |
| **LLM/AEO checklist** | Step-by-step tasks to make Polaris appear in ChatGPT/Perplexity/Gemini |
| **Google Sheets integration** | Connect your content tracker sheets to "My Content" tab |

## Setup in 4 Steps (20 minutes total)

### Step 1 — Create/update your GitHub repository

If you already have `adirispol/ORM-Dashboard`:
- Go to the repo → delete old files or replace them

If starting fresh:
1. Go to github.com → New repository
2. Name: `ORM-Dashboard` (public)
3. Initialize with README ✓

### Step 2 — Upload all files

Upload in this exact structure:
```
scripts/crawler.py        → create "scripts" folder, upload crawler.py
.github/workflows/crawl.yml  → create folders, upload crawl.yml
index.html                → upload to root
data/.gitkeep             → create empty "data" folder (add .gitkeep file)
```

**How to create .gitkeep**: In the repo, click "Add file" → "Create new file" → type `data/.gitkeep` as filename → commit.

### Step 3 — Set GitHub Secrets (for API keys)

Go to: **Repo → Settings → Secrets and variables → Actions → New repository secret**

| Secret Name | Where to Get | Monthly Cost |
|-------------|-------------|--------------|
| `YOUTUBE_API_KEY` | console.cloud.google.com → Enable YouTube Data API v3 → Create credentials | Free (10K/day) |
| `NEWSAPI_KEY` | newsapi.org/register | Free (100/day) |
| `CLAUDE_API_KEY` | console.anthropic.com | ~₹300-500/month |

> **Without these keys:** Crawler still works. Reddit + Google News RSS + Bing News RSS + DDG search all work with zero keys. Keys just give you real YouTube stats, more news sources, and AI sentiment.

### Step 4 — Enable Actions permissions

Go to: **Repo → Settings → Actions → General → Workflow permissions**
- Select: **"Read and write permissions"** ← CRITICAL
- Check: "Allow GitHub Actions to create and approve pull requests"
- Save

### Step 5 — First manual run

Go to: **Repo → Actions → "Polaris ORM Crawler v8" → Run workflow**

Watch the logs. After ~10-15 minutes, `data/` folder will populate with JSON files.

### Step 6 — Deploy dashboard

1. Go to: **Repo → Settings → Pages**
2. Source: "Deploy from a branch"
3. Branch: `main`, Folder: `/(root)`
4. Save
5. Dashboard live at: `https://adirispol.github.io/ORM-Dashboard/`

## Google Sheets Setup (My Content Tab)

To connect your content tracker:

1. Open your Google Sheet
2. File → Share → Publish to web
3. Select each tab → format: CSV → Publish (repeat for each tab)
4. Copy Sheet ID from URL: `docs.google.com/spreadsheets/d/**[SHEET_ID]**/edit`
5. Paste in dashboard ⚙ Settings

**Expected tab names** (configurable in Settings):
- `Reddit` — Week, Questions Posted, Answers Given, Impressions
- `Quora QA` — Date, Question Link, Answer Link, Upvotes
- `Quora Comments` — Date, Drive Link, Medium Link, Comment Text
- `YouTube Comments` — Date, Comment Text, Likes
- `Newspaper` — Date, Article Title, Published Count, Impressions
- `Medium` — Date, Article Title, URL, Reads

## YouTube Deep Comment Crawl

The crawler does a **checkpoint-based comment crawl** that:
- Processes 6 videos per run (every 30 min)
- Full crawl of all Polaris-related videos completes in ~5-6 days
- Stores ONLY comments mentioning "polaris" or "pst"
- Accumulates data over time in `data/yt_checkpoint.json`

Check progress: Dashboard → Debug tab → "YouTube Comment Crawl Progress"

**Requires `YOUTUBE_API_KEY`** — without it, only video metadata is fetched via search fallback.

## Brand Presence Score (BPS)

The BPS is a **0–100 marketing manager metric** designed to answer: *"How visible is Polaris online?"*

| Grade | Score | Meaning |
|-------|-------|---------|
| A+ | 80–100 | Excellent. Multi-platform positive narrative. |
| A | 65–79 | Strong presence. Minor gaps on some platforms. |
| B | 50–64 | Decent but patchy. Need consistent seeding. |
| C | 35–49 | Growing but not enough. Increase activity. |
| D | 0–34 | Brand barely visible. Urgent action needed. |

**Formula weights:**
- News/PR × 3.0 (highest credibility)
- Quora × 2.5 (high intent, Google-ranked)
- Medium × 2.0 (SEO value)
- YouTube × 2.0 (video reach)
- Reddit × 1.5 (peer trust)
- Portals × 1.0–1.2 (college directories)

Positive sentiment = 1.3× bonus | Negative = 0.3× penalty | High impressions = up to 2× bonus

## Troubleshooting

**"Only news articles showing, nothing else"**
→ This was the v6 bug (Google HTML scraping was blocked). v8 uses DDG + Bing — this is fixed.

**"Actions workflow not running"**
→ Check: Actions → General → Workflow permissions = "Read and write" ✓

**"0 results from DDG/Bing"**
→ GitHub Actions IP may be rate-limited. Check Debug tab for warnings. Crawler will retry on next run.

**"Sentiment all neutral"**
→ Add `CLAUDE_API_KEY` as a GitHub Secret. Without it, keyword-based sentiment is used (less accurate).

**"Dashboard shows old data"**
→ Click ⟳ Refresh or wait for next crawl. Data auto-refreshes every 30 minutes.
